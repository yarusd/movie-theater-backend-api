Cinematic Data & Booking API

Welcome to the MovieTime Developer Portal. This API provides an enterprise-grade cinematic infrastructure, supporting dynamic seat reservations, secure financial transactions, and advanced movie cataloging. Built for automated testing labs and high-performance booking simulations.

GET /health

System Integrity

{ "status": "UP", "moviesCount": 60, "serverTime": "2026-03-24T21:00Z" }
GET /movies

Full Movie Catalog

Retrieves all 60 movies stored in the system. Use this to verify data consistency across the platform.

[ { "id": 1, "title": "ThunderBolts*", "genre": "Action" }, ... ]
GET /movies?filters

Advanced Filtering & Search

ParamTypeExample / Description
qStringSmart search (Title, Genre, Cast). e.g. ?q=Gladiator
genreStringFilter by category. e.g. ?genre=Sci-Fi
yearIntegerFilter by release year. e.g. ?year=2024
GET /movies/:id/occupied

Occupied Seats Check

Checks availability for a specific session. **All query parameters are mandatory.**

Example Request:

GET /api/movies/25/occupied?date=2026-10-10&time=20:00

Response:

["A1", "C5", "E8"]
GET /orders/:userId

My Order History

Authorization: Use x-user-id to view your own orders. Use x-api-key to view any user's history (Admin only).

Response (200 OK):

[ { "orderId": "ORD-123", "movieId": 1, "seats": ["A1"], "status": "confirmed" } ]

POST /payments/pay-existing

Pay for Reservation

Converts a free reservation (ORD-...) into a paid status using the original Order ID.

Request Body:

{
  "orderId": "ORD-1712345678",
  "cardHolder": "Neomi Levi",
  "cardNumber": "1234123412341234",
  "expiry": "12/28",
  "cvv": "987"
}
POST /payments/checkout

Direct Paid Checkout

One-step booking and payment. Dynamic pricing: Rows A-B (₪35), C-D (₪50), E (₪65).

{
  "userId": 1, "movieId": 1, "seats": ["E1"],
  "cardHolder": "Adam Norado", "cardNumber": "1234567812345678",
  "expiry": "12/28", "cvv": "123", "date": "2026-10-10", "time": "20:00"
}
PUT /movies/:id

Update Movie Content

Requires x-api-key header for authorization.
