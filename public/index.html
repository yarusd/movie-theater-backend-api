🎥 MovieTime API Documentation

Base URL: https://movie-time-api.onrender.com/api

🔐 Authentication & Security

Operations marked with API KEY require the header: x-api-key: [YOUR_PRIVATE_KEY]

🟢 GET - Retrieval Requests

GET /health

System Status

Check if the server is live and get database metrics.

Response:

{ "status": "UP", "moviesCount": 60, "serverTime": "2026-03-24T21:49:12Z" }
GET /movies

Get All Movies

Returns the full catalog of movies stored in the system.

GET /movies?filters

Movie Search & Filtering

Search movies using query parameters. The q parameter searches both titles and descriptions.

ParameterTypeExample
idInteger/movies?id=25
qString/movies?q=Gladiator
genreString/movies?genre=Action
yearInteger/movies?year=2024

🟡 POST - Create Requests

POST /register

User Registration

Create a new account. Name must be letters only. Password: min 6 chars.

Request Body:

{
  "name": "Neomi Levi",
  "email": "neomi@example.com",
  "password": "password123",
  "password_confirmation": "password123"
}

Response (201 Created):

{ "message": "User registered successfully", "userId": 1711234567 }
POST /login

Authentication (Login)

Mandatory fields: email, password.

Request Body:

{ "email": "user1@test.com", "password": "123456" }

Test Credentials:

EmailPasswordRole / Result
admin@test.comadmin123Admin (200 OK)
user1@test.com123456User 1 (200 OK)
user2@test.com123456User 2 (200 OK)
locked@test.com123456Locked (403 Forbidden)

Response (200 OK):

{ "id": 1, "name": "Alice Cohen", "email": "user1@test.com", "role": "user" }
POST /movies API KEY

Create New Movie

Mandatory fields: title (2-60 chars), genre, duration (number).

Request Body:

{
  "title": "Gladiator II",
  "genre": "Action",
  "duration": 148
}

Response (201 Created):

{
  "id": 1711234888,
  "title": "Gladiator II",
  "genre": "Action",
  "duration": 148
}
POST /orders

Book Tickets

Book up to 8 tickets per order. Requires valid userId and movieId.

Request Body:

{ "userId": 1, "movieId": 25, "quantity": 3 }

Response (201 Created):

{ 
  "orderId": "ORD-1711234567", 
  "status": "confirmed", 
  "message": "Tickets for 'Gladiator II' booked successfully!" 
}

🔵 PUT - Update Requests

PUT /movies/:id API KEY

Modify Movie Data

Update fields like rating or description for a specific movie ID.

Response:

{ "message": "Movie updated successfully", "updatedFields": { "rating": 9.5 } }

🔴 DELETE - Removal Requests

DELETE /movies/:id API KEY

Remove Movie

Permanently delete a movie entry by its ID.

Response:

{ "message": "Movie deleted successfully" }
DELETE /test/reset API KEY

Database Reset

QA Tool: Resets movies and users to original state (60 movies + 4 original users).

Response:

{ "message": "Database reset to initial state successfully." }
