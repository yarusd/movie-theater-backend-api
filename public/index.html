🔑 Authentication & Headers

Protecting our resources is a priority. Please include the following headers in your requests to access restricted data:

x-api-key

Required for Admin endpoints (POST/PUT/DELETE movies).

master-key

x-user-id

Required for personal data safety (Orders). Prevents IDOR violations.

{user_id}

📊 HTTP Status Standards

CodeLabelQA Scenario Description
200OKSuccessful data retrieval or update.
201CreatedNew Resource (Movie/Order/User) created successfully.
400Bad RequestValidation failed. Check English characters or mandatory fields.
401UnauthorizedMissing or invalid x-api-key header.
402Payment Req.Transaction declined. Card expired or insufficient funds.
403ForbiddenLocked account or IDOR security attempt detected.
404Not FoundThe requested Resource ID does not exist in the database.
409ConflictDuplicate movie title or seats already booked.

GET /movies?filters

Advanced Filtering OR Dynamic Sorting

Use ?q= for global search, or specific keys for precision. Note: Sorting and Filtering are mutually exclusive and cannot be combined.

ParamValuesQA Target
?q=Any textGlobal Search: Scans Title, Genre, Cast.
?title=Movie NameSearch only in movie titles.
?cast=Actor NameSearch only in actor names.
?genre=Genre NameFilter by exact genre (Action, Horror...).
?sort=rating, year, titleSorts entire database (high to low).

POST /register

User Account Creation

Rules: English letters only. Name: 2-50 chars. Password: Min 6 chars.

Request Body:

{
  "name": "Neomi Levi",
  "email": "neomi@test.com",
  "password": "securepass123",
  "password_confirmation": "securepass123"
}

Response (201 Created):

{ "message": "User registered successfully", "userId": 881234 }

Role / Status Email Password Expected Result

POST /orders
Create Reservation

Free Reservation

Create a temporary seat booking. Note: ticketCount must match seats array length.

Request Body:

{
  "userId": 1,
  "movieId": 25,
  "ticketCount": 2,
  "seats": ["B1", "B2"],
  "date": "10/10/2026",
  "time": "20:00"
}

Response (201 Created):

{ 
  "orderId": "ORD-17112345", 
  "status": "confirmed",
  "message": "Reservation created successfully!",
  "userId": 1,
  "movieId": 25,
  "ticketCount": 2,
  "seats": ["B1", "B2"],
  "date": "10/10/2026",
  "time": "20:00"
}
POST /payments/pay-existing

Payment for Reservation

Convert an existing unpaid reservation (ORD-...) into a confirmed paid order.

Request Body:

{
  "orderId": "ORD-17112345",
  "cardHolder": "Adam Norado",
  "cardNumber": "4580123456789012",
  "expiry": "12/28",
  "cvv": "123"
}

Response (200 OK):

{
  "message": "Reservation successfully converted to paid order!",
  "order": {
    "orderId": "ORD-17112345",
    "userId": 1,
    "movieId": 25,
    "seats": ["B1", "B2"],
    "status": "confirmed",
    "paymentStatus": "paid",
    "totalAmount": "₪100.00",
    "transactionId": "TXN-PAY-A1B2C3D4"
  }
}
