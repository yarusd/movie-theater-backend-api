🎥 MovieTime API Documentation

Base URL: https://movie-time-api.onrender.com/api

🔐 Authentication & Security

1. Administrative operations API KEY: Require the header: x-api-key: [YOUR_PRIVATE_KEY]

2. User Identity USER ID: To prevent IDOR, user-specific requests require the header: x-user-id: [USER_ID] to verify ownership.

🟢 GET - Retrieval Requests

GET /health

System Status

Check if the server is live and get database metrics.

Response (200 OK):

{ "status": "UP", "moviesCount": 60, "serverTime": "2026-03-24T21:49:12Z" }
GET /movies

Get All Movies

Returns the full catalog of movies stored in the system.

GET /movies?filters

Movie Search, Filtering & Sorting

Search movies using query parameters. Support for partial words and combined filters.

ParameterTypeExample / Description
idInteger/movies?id=25
qStringGlobal Search: Searches by Title, Genre, or Cast. Partial words supported (e.g. ?q=thun).
genreStringFilter by genre. Case-insensitive. Must be a valid movie category (e.g., Action, Sci-Fi, Horror).
yearInteger/movies?year=2024
badgeStringFilter by tab: NOW SHOWING, COMING SOON, or EXTENDED RUN.
sortStringUse sort=rating to get movies from highest to lowest rating.
GET /movies/:id/occupied

Seat Availability Check

Check which seats are already taken for a specific movie, date, and time.

Query Params: date (YYYY-MM-DD), time (HH:mm)

Response (200 OK):

["A1", "A2", "C5"]
GET /orders/:userId USER ID

User Order History

Retrieves all bookings for a user. Requires x-user-id matching the :userId to prevent IDOR.

🟡 POST - Create Requests

POST /register

User Registration

Create a new account. Name: letters only. Password: min 6 chars.

Request Body:

{
  "name": "Neomi Levi",
  "email": "neomi@example.com",
  "password": "password123",
  "password_confirmation": "password123"
}

Response (201 Created):

{ "message": "User registered successfully", "userId": 1711234567 }
POST /login

Authentication (Login)

Mandatory: email, password.

Test Credentials:

EmailPasswordRole / Result
admin@test.comadmin123Admin (200 OK)
user1@test.com123456User (200 OK)
locked@test.com123456Locked (403 Forbidden)

Response (200 OK):

{ "id": 1, "name": "Alice Cohen", "email": "user1@test.com", "role": "user" }
POST /movies API KEY

Create New Movie

Mandatory: title, genre, duration, year, badge.

Note: Returns 409 Conflict if movie title already exists.

Request Body:

{
  "title": "Gladiator II",
  "genre": "Action",
  "duration": 148,
  "year": 2024,
  "badge": "NOW SHOWING"
}

Response (201 Created):

{ "id": 1711234888, "title": "Gladiator II", ... }
POST /orders

Book Tickets (Seats Selection)

Book specific seats (A1-E8). Max 8 per order. Date must be in the future.

Request Body:

{ 
  "userId": 1, 
  "movieId": 1, 
  "seats": ["A1", "A2"],
  "date": "2026-12-25",
  "time": "19:00"
}

Response (201 Created):

{ "orderId": "ORD-1711234567", "status": "confirmed", ... }

🔵 PUT - Update Requests

PUT /movies/:id API KEY

Modify Movie Data

Full update of movie details. Note: Returns 409 Conflict if updating to an existing title.

Response (200 OK):

{ "message": "Movie updated successfully", "movie": { ... } }

🔴 DELETE - Removal Requests

DELETE /movies/:id API KEY

Remove Movie

Permanently delete a movie by ID.

Response (200 OK):

{ "message": "Movie deleted successfully" }
DELETE /orders/:orderId

Cancel Order

Removes a booking and frees the seats for future use.

Response (200 OK):

{ "message": "Order cancelled successfully. Seats are now free." }
DELETE /test/reset API KEY

Database Reset (QA Tool)

Resets movies, users, and clears all orders to initial state.

Response (200 OK):

{ "message": "Database reset successfully." }

404Not FoundThe movie or resource does not exist.
