🎥 MovieTime API Documentation

Base URL: https://movie-time-api.onrender.com/api

🔐 Authentication & Security

Operations marked with API KEY require the following header:

x-api-key: [YOUR_PRIVATE_KEY]

🟢 GET - Retrieval Requests

GET /health

System Status

Monitors server availability and returns database statistics.

Response:

{ "status": "UP", "moviesCount": 60, "serverTime": "2026-03-24T21:49:12Z" }
GET /movies

Get All Movies

Fetches the complete library of movies currently stored in the system.

🟡 POST - Create Requests

POST /register

User Registration

Register a new account. Name must contain only letters. Password: 6-20 characters.

Request Body:

{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "password123"
}

Response:

{ "message": "User registered successfully", "userId": 1711234567 }
POST /login

Authentication (Login)

Verify credentials to receive a user profile. Mandatory fields: email, password.

EmailPasswordExpected Result
admin@test.comadmin123Admin (200 OK)
user1@test.com123456User (200 OK)
locked@test.com123456Locked (403 Forbidden)

Response (200 OK):

{ "id": 3, "name": "Admin User", "email": "admin@test.com", "role": "admin" }
POST /movies API KEY

Create New Movie

Add a new entry to the catalog. Mandatory fields: Title (max 60), Genre, Duration.

Response:

{ "id": 1711234888, "title": "Gladiator II", "genre": "Action", "duration": 148 }

🔵 PUT - Update Requests

PUT /movies/:id API KEY

Modify Movie Data

Update existing metadata using its unique ID.

Response:

{ "message": "Movie updated successfully", "updatedFields": { "rating": 9.2 } }

🔴 DELETE - Removal Requests

DELETE /movies/:id API KEY

Remove Movie

Permanently delete a record using its ID.

Response:

{ "message": "Movie deleted successfully" }
DELETE /test/reset API KEY

Database Reset

QA tool to reset the database to its initial state for fresh testing sessions.

Response:

{ "message": "Database reset to initial state successfully." }
