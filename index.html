🎥 MovieTime API Documentation

Project by Yarus and Tzedek

Base URL: https://movie-time-api.onrender.com/api

🔐 Authentication & Security

Operations marked with API KEY require the following header:

x-api-key: [YOUR_PRIVATE_KEY]

🟢 GET - Retrieval Requests

GET /health

System Health Status

Verifies if the server is live and provides an overview of the current database size and server timestamp.

GET /movies

Retrieve All Movies

Fetches the complete library of movies currently stored in the system. This returns an array of movie objects.

GET /movies?filters

Dynamic Filtering & Searching

Narrow down results by applying one or more query parameters to the URL. This allows for precise data retrieval.

Query Parameter Type Usage Description
id Integer Filter by specific unique Movie ID
q String Global search keywords across titles and descriptions
genre String Filter by category (e.g., Action, Horror, Sci-Fi)
year Integer Filter by the exact movie release year

Example: /movies?genre=Action&year=2024

🟡 POST - Create Requests

POST /register

User Registration

Register a new user account in the system database.

Request Body: {
  "name": "Tzedek Silver",
  "email": "tzedek.silver@example.com",
  "password": "securePassword"
}

Response:

{
  "message": "User registered successfully",
  "userId": 1711234567
}
POST /login

Authentication (Login)

Verify user credentials and retrieve profile details.

Request Body: { "email": "admin@test.com", "password": "admin123" }

Response:

{
  "id": 3,
  "name": "Admin User",
  "email": "admin@test.com",
  "role": "admin"
}
POST /movies API KEY

Create New Movie

Add a new entry to the catalog. Data is validated via Joi Schema to ensure integrity.

Request Body: {
  "title": "Gladiator II",
  "genre": "Action",
  "year": 2024,
  "duration": 148
}

Response:

{
  "id": 1711234888,
  "title": "Gladiator II",
  "genre": "Action",
  "year": 2024,
  "isFeatured": false
}
POST /orders

Book Tickets (Orders)

Process a new ticket reservation for a specific movie screening.

Request Body: {
  "movieId": 2,
  "seats": ["C3", "C4"],
  "totalPrice": 90,
  "customerEmail": "user@test.com"
}

Response:

{
  "orderId": "ORD-99210",
  "status": "confirmed",
  "message": "Tickets booked successfully"
}

🔵 PUT - Update Requests

PUT /movies/:id API KEY

Modify Movie Data

Update existing movie metadata using its ID in the URL path.

Request Body Example: { "rating": 9.2, "isFeatured": true }

Response:

{
  "message": "Movie updated successfully",
  "updatedFields": { "rating": 9.2, "isFeatured": true }
}

🔴 DELETE - Removal Requests

DELETE /movies/:id API KEY

Remove Movie

Permanently delete a movie record from the database using its unique ID.

Response:

{ "message": "Movie deleted successfully" }
DELETE /test/reset API KEY

Database Reset (QA Tool)

Clears all dynamic data (orders and seat maps) for fresh automated testing sessions.

Response:

{ "success": true, "message": "Database reset completed" }

📊 Response Status Standards

Status Code Label Description
200 / 201 Success Operation completed successfully.
400 Bad Request Validation failed. Check your request body or schema.
401 Unauthorized Invalid or missing x-api-key header.
404 Not Found The requested resource or ID does not exist.
500 Server Error Internal failure. The system logs have recorded the error.
